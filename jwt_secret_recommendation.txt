# Рекомендации по настройке JWT_SECRET

## Проблема
Обнаружена ошибка аутентификации: "JsonWebTokenError: invalid signature"
Это означает, что секретный ключ, используемый для проверки JWT-токена, не соответствует ключу, который использовался для создания токена.

## Решение
1. Убедитесь, что в файле .env на бэкенде установлен правильный JWT_SECRET:

```
JWT_SECRET=ваш_секретный_ключ
```

2. Важно использовать один и тот же секретный ключ для:
   - Создания токена в routes/auth.ts
   - Проверки токена в middlewares/authenticate.ts
   - Проверки токена в middlewares/authRole.ts

3. После изменения JWT_SECRET:
   - Перезапустите бэкенд-сервер
   - Выйдите из системы на фронтенде
   - Войдите заново, чтобы получить новый токен

## Проверка
Текущая длина JWT_SECRET: 14 символов
Если вы не уверены, какой секретный ключ использовался при создании токена, рекомендуется установить новый секретный ключ и заново войти в систему.
